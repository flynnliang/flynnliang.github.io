Spring Data JPA

发表于 2016-12-05 | 分类于 框架

Spring-Data-JPA简介

数据持久层的基本CRUD操作，只声明下面一个接口就搞定，这就是Spring-Data-JPA

import org.springframework.data.jpa.repository.JpaRepository;

public interface UserDAO extends JpaRepository<User, String> {
}

没有Impl，没有SQL，没有XML，就是那么懒！！

它是懒人利器的理由:

  • 基本CRUD操作，它有了：

    public interface JpaRepository<T, ID extends Serializable>
    extends PagingAndSortingRepository<T, ID>, QueryByExampleExecutor<T> {

        List<T> findAll();

        List<T> findAll(Sort sort);

        List<T> findAll(Iterable<ID> ids);

        <S extends T> List<S> save(Iterable<S> entities);

        void flush();

        <S extends T> S saveAndFlush(S entity);

        void deleteInBatch(Iterable<T> entities);

        void deleteAllInBatch();

        T getOne(ID id);

        @Override 
        <S extends T> List<S> findAll(Example<S> example);

        @Override
        <S extends T> List<S> findAll(Example<S> example, Sort sort);
}

  • 通过规范的方法名解析创建查询

    public interface UserDAO extends JpaRepository<User, String> {
    //通过用户名查找用户
    User findByName(String name);
    //通过用户名和密码查询用户
    User findByNameAndPassword(String userName, String password);
}

  • 通过相应注解创建个性化查询

    @Modifying 
@Transactional
 @Query("update user a set a.passwd = ?1 where a.uuid = ?2") 
 public int updateUserPasswd(String ps, String uuid);

当然还有其他的，只是目前未深入使用。

JPA是一种ORM(Object Relation Mapping)框架,全称是Java Persistence API,可以通过@注解或者XML方式，实现对象与数据库表间的映射，将运行期的对象持久化到数据库中，Spring的生态圈不可能不把这家伙吸纳了。

Sring-Data-JPA能帮我们偷啥懒

Mybatis(Ibatis)和hibernate或原生的JDBC相信Java程序员都熟悉不少，不管它们多优秀，开发者都难免都还摆脱不了SQL编写或者持久层的业务逻辑处理这块敲不少代码。在此不是要挑起战争，只能说着重点不同，纯粹是在懒人的角度看待Spring-Data-JPA，它让我少了不少事:

  • 不需要费精力写sql
  • 不需要关注EntityManager 的创建
  • 不需要关注事物处理
  • 不需要关注持久层核心业务处理

你的持久层方面没了SQL 没有XML，你需要做的声明持久层接口并选择性集成它的接口，搞定解决一切，是不是懒得可以。

Spring-Data-JPA 咋用

为啥它能那么强悍，我脑容量很小，对待框架，只有出现莫名其妙Bug、Boss大发慈悲的时候有空发呆，这两种情况才会去看源码，所以平时会用，用得好足矣。

根基–数据库Model

既然不需要SQL语句或XML文件进行映射，必然会有自己的方法进行Model与数据库表之间进行对应。Spring-Data-JPA 使用的是注解的方式 + Model属性名称与字段名称对应两种方式进行映射，如下下面是一般基本会有的，使用到的时候自行查阅官网API即可:

@Entity   
@Inheritance
@DiscriminatorColumn(name = "type")
DiscriminatorValue("user")
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(generator = "IDGenerator")
    @GenericGenerator(name = "IDGenerator", strategy = "uuid")
    private String uuid;

    @Column(name = "type", nullable = false, insertable = false, updatable = false)
    private String type;

此处省略好.............

}
  • @Entity : Specifies that the class is an entity //表明此为实体类
  • @Inheritance : Defines the inheritance strategy to be used for an entity class hierarchy. It is specified on the entity class that is the root of the entity class hierarchy. //实体类之间的继承策略，其中若标记在则表明该实体类为父(根)类
  • @DiscriminatorColumn(name = “type”) //区分字段，name为字段名称
  • @DiscriminatorValue //区分值
  • @Table(name = “users”) //表名称
  • @Id //数据库表唯一标示ID
  • @GeneratedValue(generator = “IDGenerator”) 自动生成(生成策略为ＩＤ生成器)
  • ＠Column　//当字段名称与实体属性名称不一致时，使用该注解标示
